# Planner 学習データ: 入力仕様と損失（P1 主・P2 予告）

本稿は `design_docs/planner_model_flow_matching_architecture.md` の **P1 着手・ルール選択・高忠度 BEV** 方針に沿い、**学習・推論で固定したい入力テンソルの形**と **損失の定義**を確定させるためのメモです。実装時はここを「契約」としてバージョン管理し、変更したら本書を更新してください。

---

## 1. スコープと用語

| 用語 | 意味 |
|------|------|
| **サンプル** | ある時刻 `t` における 1 フレーム（または短い過去スタック）から、未来 `t+Δ … t+H` を予測する単位 |
| **教師軌道** | ロギングされた将来姿勢または参照経路（MPC 入出力・Planner 出力など）から得た **ego 座標系**の `(x, y)` 系列 |
| **モード id** | 多峰教師に対し、各サンプルが属するクラスタ `m ∈ {0,…,K-1}`（オフライン付与） |

**学習に含めないもの（方針）**: 推論時の **ルールスコアによる候補選択**は **勾配なしの後段**であり、P1 の主損失には **入れない**（将来、必要なら「選ばれた軌道への蒸留」は別タスクとして検討）。

---

## 2. 主入力: BEV テンソル（`bev_scene_stack` 準拠）

### 2.1 ソースとメッセージ

- **トピック（既定）**: `bev_scene_stack/tensor`（`std_msgs/Float32MultiArray`）
- **レイアウト**: `layout.dim` は channel-major（`C × H × W` を C 順に flatten）に合わせる。  
  - `dim[0]`: `label=channel`, `size=C`, `stride=H*W`  
  - `dim[1]`: `label=height`, `size=H`, `stride=W`  
  - `dim[2]`: `label=width`, `size=W`, `stride=1`

### 2.2 形状（現行パラメータからの契約値）

`bev_scene_stack.param.yaml` の既定に基づく **契約**（パラメータを変えたら本表を更新すること）。

| 項目 | 値 |
|------|-----|
| `C`（チャネル数） | **4** |
| `grid_x_min`, `grid_x_max` | -12.0, 52.0（車体前方 x 正） |
| `grid_y_min`, `grid_y_max` | -18.0, 18.0（左 y 正） |
| `resolution` | 0.25 m |
| `H` | `ceil((x_max - x_min) / res)` = **256** |
| `W` | `ceil((y_max - y_min) / res)` = **144** |

**学習用テンソル形状**: `X_bev ∈ ℝ^{B×C×H×W}`、`B` はバッチ。`C=4` の **チャネル意味（固定）**:

| index | 意味 |
|------:|------|
| 0 | レーン（Lanelet OSM ラスタ） |
| 1 | 自車参照経路（現行 Planner の trajectory チャネル） |
| 2 | 障害物（`/aichallenge/objects` + V2X 相手車フットプリント） |
| 3 | 自車 bbox |

### 2.3 前処理・正規化（推奨の既定）

学習・推論で **同一パイプライン**を通す。

1. **reshape**: `Float32` 列 `→` `float32` の `(C, H, W)`。バッチでは `(B, C, H, W)`。
2. **値域**: 現状チャネルはおおむね `{0,1}` の二値ラスタ。**既定の正規化**は  
   `x̃_c = clip(x_c, 0, 1)` のみ（チャネル別 mean/std はオプション。統計が安定したら `μ_c, σ_c` を学習データから推定してもよい）。
3. **データ拡張（保守的）**: 高忠度 BEV を活かすため、**Yaw 回転・平行移動を BEV に直接かけない**（幾何が崩れる）。必要なら **軌道教師のみ**に小さなノイズ、または **別ドライバのログ**で多様性を確保。
4. **欠損**: BEV が無いフレームは **サンプル棄却**（マスク学習は P2 以降で検討）。

### 2.4 時間整合

- **同期**: 教師軌道の時刻原点 `t` と、BEV の `header.stamp`（または収録メタの `t_bev`）の差が **許容閾値**（例: ≤ 50 ms）を超えるサンプルは棄却または下サンプル。
- **将来ホライズン**: `T` ステップ、間隔 `Δt`（例: MPC 周期に合わせ 0.05 s × `T=40` = 2 s）は **データセットのメタ**として固定し、軌道教師を **等間隔リサンプル**して `(T, 2)` に揃える。

## 3. 補助入力（ベクトル）

BEV 以外は **任意だが推奨**するベクトル `z_aux`（例: `ℝ^{D}`）。P1 では次を **契約候補**とする（実装で削ってよい）。

| ブロック | 内容（例） | 次元の目安 |
|----------|------------|-------------|
| **Ego 運動** | `v_x, v_y, ω`（`kinematic_state` から） | 3 |
| **ヨー（相対）** | map 対 base の yaw `ψ`（教師と整合する座標系で） | 1 |
| **スカラー目的** | ラップ残り、セクター index など正規化スカラー | 1〜4 |

**結合**: `z = concat(flatten(X̃_bev), z_aux)` にせよ、`Encoder(X̃_bev)` の出力に `MLP(z_aux)` を加算・concat せよ、は実装任せ。**YAML で次元一覧を固定**すること。

---

## 4. 教師信号（出力ターゲット）

### 4.1 軌道表現

- **座標系**: **ego / base_link**、原点 = BEV と同じ自車基準。
- **形状**: `Y ∈ ℝ^{B×T×2}`。チャネルは `(x, y)`。`z` は教師から落とすか、地面 `z≡0`。
- **リサンプル**: ログは可変ステップでもよいが、学習前に **弧長または等時間**で `T` 点に揃える。

### 4.2 P1 用: モード id（多峰教師）

オフラインで、データセット全体またはセグメント単位で:

1. 各サンプルの **終端変位** `(Δx_T, Δy_T)` や **全軌道ベクトル**（`flatten(Y)`）に対し **k-means（K クラスタ）** を実行する、または
2. **走行ラベル**（イン／アウト／オーバーテイク等）が取れるなら、その離散ラベルを `m` とする。

**サンプルごとに** `m ∈ {0,…,K-1}` を付与し、学習時は **対応するヘッド `head_m` だけ**に勾配を流す（下記 §5.1）。

---

## 5. 損失関数（フェーズ別）

### 5.1 P1（K 並列ヘッド + 教師モード）— **主損失**

**設定**: ネットは `K` 個の軌道候補 `Ŷ_k ∈ ℝ^{B×T×2}` を出力（または共有 trunk + `K` 個の浅い head）。

**ハード割当（既定）**: サンプル `i` のモード `m_i` に対し

\[
\mathcal{L}_{\mathrm{pose}} = \frac{1}{BT} \sum_{i,b,t} \phi\bigl( \hat{y}^{(i)}_{m_i, b, t} - y^{(i)}_{b,t} \bigr)
\]

- `φ`: **Smooth L1 (Huber)** を既定（外れ値に強い）。`δ` は 0.1 m 程度からチューニング。
- **勾配**: `k ≠ m_i` のヘッドには **この損失からは勾配を流さない**（他ヘッドは下記オプションのみ）。

**オプション（推奨度・中）**: **未割当ヘッドの沈黙ペナルティ**（全ヘッドが同じ解に崩れるのを防ぐ）

\[
\mathcal{L}_{\mathrm{div}} = \lambda_{\mathrm{div}} \cdot \frac{1}{K(K-1)} \sum_{k \neq k'} \mathrm{mean}_{b,t} \bigl[ \max(0, d_{\min} - \| \hat{y}_{k,b,t} - \hat{y}_{k',b,t} \| )^2 \bigr]
\]

- `d_min`: 格子解像度の数倍（例: 0.5〜1.0 m）。`λ_div` は小さく始める（例: 0.01）。

**オプション（推奨度・低）**: **曲率ペナルティ**（教師なしの幾何正則化）

\[
\mathcal{L}_{\kappa} = \lambda_\kappa \sum_{k} \mathrm{mean}_t \bigl[ (\Delta^2 x_{k,t})^2 + (\Delta^2 y_{k,t})^2 \bigr]
\]

二階差分（離散曲率の代理）。**教師が滑らかなら** `λ_κ` は極小でよい。

**P1 総損失（既定）**

\[
\mathcal{L}_{\mathrm{P1}} = \mathcal{L}_{\mathrm{pose}} + \mathcal{L}_{\mathrm{div}} + \mathcal{L}_{\kappa}
\]

（未使用項は係数 0。）

### 5.2 P1 で「やらない」こと（明示）

- **ルールスコアを損失に直接加算しない**（推論時のみ使用）。学習と評価の責務分離。
- **BEV 再構成損失**（encoder-decoder で BEV を復元）は行わない（BEV は入力のみ）。

### 5.3 P2（Flow Matching）— **予告（契約草案）**

教師 `Y` を **ノイズ `ε` から線形 OT で結ぶ**最簡形（rectified / linear conditional flow の特別場合）を想定。

- サンプルごとに `ε ~ 𝒩(0, I)`（形状 `Y` と同じ）。
- 時刻 `τ ∈ [0,1]` を一様サンプル。線形橋: `Z_τ = (1-τ) ε + τ Y`。
- 速度場の教師: `V^* = Y - ε`（`τ` に依らない定数場）。
- ネット `v_θ(Z_τ, τ, c)` に対し **Flow matching 損失**:

\[
\mathcal{L}_{\mathrm{FM}} = \mathbb{E}\bigl[ \| v_\theta(Z_\tau, \tau, c) - (Y - \epsilon) \|^2 \bigr]
\]

**K 本**: `K` 条件付け（モード埋め込み・goal token）を入れ、`ε` も **K 本独立**にサンプルし、各枝で上式。**選択は依然オフラインのモード id または推論時ルールのみ**とし、P2 でも **学習型スコアラーは必須にしない**。

（より厳密な OT-CFM や二次コスト付き flow は P3 以降で検討。）

## 6. バッチ化とマスク

| マスク | 用途 |
|--------|------|
| `valid_bev` | BEV が閾値以上の情報量（例: lane チャネル和 > ε）を満たすサンプルのみ `1` |
| `valid_traj` | 教師軌道が短すぎる・NaN のサンプルを `0` |
| **損失**: `L = sum(mask * per_step_loss) / sum(mask)` |

---

## 7. データセット行のスキーマ（推奨）

各サンプルを 1 行（または `.npz` 1 ファイル）にまとめる場合の **必須キー**:

| キー | dtype | shape |
|------|-------|-------|
| `bev` | float32 | `(4, H, W)` |
| `traj_gt` | float32 | `(T, 2)` |
| `mode_id` | int64 | scalar |
| `stamp_ns` | int64 | scalar（同期検証用） |
| `aux` | float32 | `(D,)` 任意 |

**任意**: `session_id`, `map_version`, `driver_id`（層化サンプリング用）。

---

## 8. 変更管理

- `bev_scene_stack` の **格子範囲・解像度・チャネル数**を変えたら、**§2.2 と学習コードの reshape**を同時更新する。
- 損失の係数 `λ_*` は **YAML で集中管理**し、実験 ID とコミットハッシュをログに残す。
